# Case Study: AI-Steered Optimization

This document demonstrates the core methodology used in this project: **Iteration over Manual Correction.** Instead of fixing sub-optimal code by hand, we refine the "Agentic Persona" and "System Prompt" to produce professional-grade results automatically.

## Scenario: Dockerfile Optimization

**Objective:** Generate a production-ready Dockerfile for a Python/FastAPI project.

### 1. Initial AI Output (Sub-optimal)
When prompted without specific agentic constraints, a standard LLM often produces a "functional but risky" Dockerfile:

```dockerfile
FROM python:3.12
COPY . /app
WORKDIR /app
RUN pip install -r requirements.txt
CMD ["python", "src/main.py"]
```

**Issues Identified:**
- [Security] Runs as `root` user.
- [Performance] Uses a large base image (unnecessarily large attack surface).
- [Best Practices] No multi-stage build (build dependencies included in final image).

---

### 2. The AI-Steering Intervention
Instead of editing the Dockerfile, we updated the **Senior Infra Architect** persona in `AGENTS.md` and refined the `prompts/system_devops.md` to strictly enforce operational directives.

**Refinement Instruction Added:**
> *"MANDATORY multi-stage builds. Process MUST run as a non-root user (UID >= 1000). Always use -slim or -alpine variants for runtime images."*

---

### 3. Professional Result (Post-Steering)
After the prompt refinement, the **same AI** produced this high-quality, production-ready artifact:

```dockerfile
# Stage 1: Build dependencies
FROM python:3.12-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

# Stage 2: Production runtime
FROM python:3.12-slim
WORKDIR /app

# Security: Create non-root user
RUN useradd -m -u 1001 appuser
USER appuser

# Copy only installed packages and source code
COPY --from=builder /root/.local /home/appuser/.local
COPY . .

# Environment setup
ENV PATH=/home/appuser/.local/bin:$PATH \
    PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1

CMD ["python", "src/main.py"]
```

## Conclusion
By steering the AI through architectural constraints rather than manual line edits, we ensure that **all future artifacts** generated by the server maintain this high standard. This approach reduces technical debt and aligns with the **AI-First Developer** mindset.
